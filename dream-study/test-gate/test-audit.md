# Test Audit — Phase 0a

> Generated by the Phase 0a agent. Read-only sweep across miette/, hecks/spec, hecks/rust, hecks/parity, hecks_conception/tests. Output is the worklist for sub-phases 0b–0g.

## Scope

The audit classifies every test file that touches the **consciousness pipeline** as it exists today (what becomes Body / Witness / Mind / Dream / Lucidity / SleepCycle after the refactor). Adjacent surfaces that the refactor will brush — heartbeat, organs, doctor, statusline, dispatch wrapper — are included. Surfaces that won't move (validator tests, hecksagon DSL, IDE specs) are skipped.

Classification per file :

- **current** — runs cleanly today, asserts a contract that survives the refactor without test edits
- **stale** — contract is still valid but mechanical drift (aggregate name, command name, attribute) will need a sweep when Body lands
- **broken** — assertion no longer matches behaviour (a regression we missed) — must be repaired before the suite can claim "green"
- **redundant** — covered by another test or testing a vestigial code path that retires
- **gap** — no file exists yet (used in the coverage analysis)

Test counts are `^[[:space:]]*test[[:space:]]+"` matches in `.behaviors`, `it "..."` matches in rspec, `check`/`note_pass`/`fail` matches in shell smokes. Approximate where the shape is loose.

## Coverage analysis per Phase 0 sub-phase

### 0b — full 8-cycle sleep end-to-end

> Dispatch `EnterSleep` → observe all phase transitions per-tick → cycle 8 reaches lucid REM with 8 dreams (i208 contract) → auto-wake → mood = refreshed → fatigue = rested. Recorded as (event, command, state-after) triples.

**Existing coverage**

- `body/sleep/night.behaviors` is the *closest* but tests an entirely different aggregate (`Night` with hardcoded DreamCycle1..DreamCycle8 commands) that the refactor retires. **Not reusable** — the contract being asserted (8 cycles → lucid → ended) is what 0b will assert via the new shape.
- `body/sleep/consciousness.behaviors::EnterSleep cascades` asserts emits include `["SleepEntered", "MonitoringStarted"]` — only the first transition, not the full sequence.
- `dream_content_smoke.sh` runs 10 ticks of rem_branch.sh and asserts dream_state grows ≥10 — tests *one phase* (REM), not the cycle.
- `round4_integration_smoke.sh` asserts tick advances by ≥1 over 10s — proves liveness, not phase progression.

**Gaps (total — no test exists)**

- The full per-tick sequence : EnterSleep → light → rem → deep → ... × 8 → final_light → wake. No test exists.
- The lucid-REM-on-cycle-8 contract (`dream_pulses_needed = 8` instead of 5) is referenced in `consciousness.bluebook:197` but **not tested anywhere**.
- The auto-wake-from-final-light : `CompleteFinalLight(wake_at)` should transition `state → attentive` and stamp `last_wake_at`. Not tested.
- `mood = refreshed` post-wake is implicit (mood.bluebook policy `RefreshOnFullSleep`) but not asserted in any cascade test.
- `fatigue = rested` post-wake : `Heartbeat.RecoverFatigue` is asserted in heartbeat.behaviors as a direct test (resets to alert), but not as a downstream consequence of WokenUp.

**Where new tests should land**

- `dream-study/test-gate/0b_full_night.behaviors` (or matching .rb if rspec-style) — a single behaviors file declaring the per-tick sequence. Probably long ; may need to be a shell smoke instead since it crosses many aggregates and 8 cycles' worth of transitions.
- `dream-study/test-gate/0b_full_night_smoke.sh` — driver that dispatches EnterSleep, then ticks ElapsePhase + advancement commands, asserting `(event, command, state-after)` triples per the plan. Likely 100-200 LOC. Modeled on `dream_content_smoke.sh` shape.

### 0c — cross-aggregate event chains

> `LucidRemEntered → LucidDream.BecomeLucid` ; `DeepEntered → LucidDream.EndLucidity + Heartbeat.RecoverFatigue` ; `WokenUp → WakeMood.SetWakeMood + Heartbeat.RecoverFatigue` ; `BodyPulse during awake → Awareness.RecordMoment + Heartbeat.AccumulateFatigue`

**Existing coverage**

- `body/dream/lucid_dream.behaviors::BecomeLucid cascades` and `EndLucidity cascades` test the *direct command + cross-aggregate cascade into LucidMonitor* — but they don't drive the chain *from `LucidRemEntered` event*. The trigger event is not asserted ; only that `BecomeLucid` cascades.
- `body/cycles/heartbeat.behaviors` tests `Heartbeat.RecoverFatigue` as a direct command + `AccumulateFatigue` directly — but **never as the consequence of `WokenUp` or `BodyPulse`**. The `i40` tests prove the sleep-gate-closes-on-SleepEntered policy, but that's only the gate, not the recovery.
- `pulse_fanout_smoke.sh` asserts `Tick.MindstreamTick → BodyPulse → AccumulateFatigue + PruneSynapses + ConnectNerve` end-to-end via the runtime fanout. **This is the only existing test of a cross-aggregate event chain on the pulse path.** Critical reference.
- Single-bluebook `.behaviors` runner can't follow `across` hops — every cross-aggregate chain that requires `across` is structurally untestable in `.behaviors` ; needs the cross-bluebook fanout style of `pulse_fanout_smoke.sh`.

**Gaps**

| Chain | Existing? | Gap |
